# Test fixtures

## `iphone-sample.heic`

A real iPhone-origin HEIC file used by the end-to-end test in
`../imageProcessor.test.ts` ("ensureApiSafeFormat HEIC real-fixture path").

- **Provenance:** Copied from
  `attached_assets/Screenshot_2026-03-08_at_5.13.34_PM_1773008020161.heic`,
  an iPhone screenshot uploaded to this project.
- **Format:** ISO Media, HEIF Image, HEVC Main Still Picture profile.
- **Dimensions:** 1320 × 2868 (iPhone Pro screenshot aspect ratio).
- **Size:** ~179 KB. The original task description suggested "a few KB",
  but a synthetic minimal HEIC (a few hundred bytes, generated by
  ImageMagick) was rejected during code review as not credibly
  representative of iPhone output. ~179 KB is well below typical iPhone
  HEIC photo sizes (1–3 MB) and small enough to keep the repo lean
  while still being a genuine iPhone-produced bitstream.

## Why a real iPhone HEIC

iPhone HEIC uploads are the primary real-world source of HEIC for this
service. Synthetic HEIC bitstreams (e.g. those produced by ImageMagick's
HEIF encoder) do not exercise the same encoder profiles, item layouts, or
metadata structures that ship with iPhone photos. Committing a small but
genuine iPhone HEIC ensures `heic-convert` (and the underlying libheif
build) is verified against actual iPhone output, so version upgrades or
build-flag regressions surface in CI rather than in production.
